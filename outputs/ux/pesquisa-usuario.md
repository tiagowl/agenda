# Pesquisa de Usuario - Sistema de Agendamento Web

## Objetivo da pesquisa
Entender como um usuario individual, em contexto mobile, organiza compromissos e quais friccoes impedem planejamento diario eficiente.

## Persona principal
### Persona: "Planejador Pratico"
- Usa o celular como dispositivo principal.
- Quer registrar compromissos rapidamente durante o dia.
- Precisa visualizar rotina em diferentes niveis (dia/semana/mes).
- Valoriza clareza e baixa complexidade.

## Jornada do usuario
1. Acessa sistema e faz login.
2. Cadastra compromisso com informacoes essenciais.
3. Revisa agenda no recorte diario/semanal/mensal.
4. Consulta dashboard para acompanhar status do dia.
5. Exporta/importa dados para backup quando necessario.

## Dores identificadas
- Perda de tempo em formularios longos para cadastro.
- Dificuldade em enxergar prioridades em tela pequena.
- Risco de perda de dados locais sem rotina de backup.
- Inseguranca na importacao de dados (medo de sobrescrever).
- Baixa visibilidade do que ficou pendente no dia.

## Necessidades principais
- Fluxo de cadastro em poucos passos.
- Navegacao clara entre visoes de calendario.
- Dashboard objetivo com metricas acionaveis.
- Feedback explicito em acoes criticas (salvar, excluir, importar/exportar).
- Layout e controles otimizados para toque.

## Hipoteses de UX a validar
- H1: Quick add reduz tempo medio de cadastro em pelo menos 30%.
- H2: Navegacao por abas fixas melhora descobribilidade das visoes.
- H3: Indicadores visuais de pendencia aumentam conclusao de compromissos.
- H4: Confirmacoes contextuais diminuem erros em importacao/exclusao.

## Oportunidades de melhoria
- Adicionar acao primaria fixa de "Novo compromisso".
- Usar componentes com toque confortavel (alvo minimo 44px).
- Exibir status de compromisso por cor + texto (nao apenas cor).
- Implementar fluxo assistido de importacao com preview.
